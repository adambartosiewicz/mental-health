# Values

## Purpose

What actually matters to me — not what I think *should* matter, and not what I would write on a CV. Values are used as a reference when deciding between options, when interpreting friction, and when designing experiments. A choice that violates a real value will produce real friction; a choice that violates a stated-but-not-real value will not.

## Update rules

- Values are added only when **demonstrated by behavior**, not just by aspiration. If something only ever feels important in conversation but is never acted on, it is an aspiration — note it as such.
- Distinguish:
  - **Lived value** — pattern of choices shows it matters.
  - **Aspirational value** — believed important; behavior does not yet reflect it.
  - **Inherited value** — comes from family/culture; unclear whether it's mine.
- Review every 6 months. Move items between categories as evidence warrants.
- When a value is removed or downgraded, log the reason briefly.

## Entry template

```markdown
### <Value name>

- **Category:** lived / aspirational / inherited
- **What it looks like in practice:** <concrete behaviors>
- **Evidence (or absence of evidence):** <sessions, decisions, recurring choices>
- **Tensions:** <other values this conflicts with>
- **Last reviewed:** YYYY-MM-DD
```

## Example

```markdown
### Autonomy over schedule

- **Category:** lived
- **What it looks like in practice:** Repeatedly chooses roles/projects with control over time, even at financial cost. Friction high when calendar is owned by others.
- **Evidence:** `sessions/2026-04-12-job-decision.md`, `sessions/2026-05-02-saturday-flat.md`
- **Tensions:** Tension with "tight collaboration" — needs structured solo blocks but also feels isolated by them.
- **Last reviewed:** 2026-06-09
```
